# Core Engine (다중 에이전트 매칭 최적화 지시서)

**Role**: 유저 에셋(Blue)을 템플릿(Yellow) 슬롯에 타임라인으로 조립하여 `edit_instruction.json` 편집 지시서 도출.

**Agent A 매칭 및 스코어링 강제 룰**:
- 결합 점수 비중: 시각적 일치도 50% (텍스트 임베딩 모델 코사인 유사도) + 역동성 일치도 30% (Optical Flow) + 길이 적합도 20%.
- **[Soft Penalty 룰]**: 동일한 유저 클립의 중복 할당을 원천 금지하지 않는다. 단, 이미 사용된 클립이 다음 슬롯 경쟁에 진입할 경우 최종 스코어에서 **30% 감점(페널티)**을 부여한 채 경쟁시킬 것. (어색한 컷오버 지양 및 리플레이/수미상관 유도 목적).

**Agent B 1-Hit 검증 및 루프 제한**:
- 도출된 가안 타임라인의 총합 길이가 60초 이내인지, 컷 타이밍이 템플릿의 타임스탬프와 오차 범위(±100ms) 내인지 검증할 것.
- **[무한 루프 차단]**: Agent B가 Agent A의 1등 가안을 반려(Reject)할 경우, 두 번째 조합을 재탐색(Recursion)하는 것을 엄격히 금지함. 한 번 반려되면 즉시 하단 Fallback 로직으로 강제 이관할 것.

**최후의 보루 (Fallback 강제 할당)**:
- Agent B의 1-Hit 반려가 발생하거나, 유저 소스의 최고 매칭 점수가 사용자 지정 임계값(UI 슬라이더 값) 미만일 경우 즉시 발동.
- 시각적 내용(Semantic) 매칭 로직을 전면 포기하고 폐기할 것.
- 오로지 **오디오 리듬(Motion & Beat)** 팩터만을 기준으로 삼아 남은 세그먼트들을 강제 할당할 것. "내용은 틀려도 템포/리듬감은 완벽한" 결과물을 무조건 반환해야 함.
