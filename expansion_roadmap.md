# 비디오 Fallback 매칭 엔진 - 확장 가능성 및 스케일업 로드맵
> 파일명: `expansion_roadmap.md`
> 목적: MVP 단계 이후 시스템이 취할 수 있는 기술적, 비즈니스적 확장 경로 명세

---

## 1. 비즈니스 모델 (BM) 확장 루트

### 1-1. B2C 숏폼 자동 생성 서비스 (MVP)
* **형태:** 웹/앱 기반의 엔드유저 대상 서비스.
* **구조:** 사용자가 유행하는 숏폼 URL과 자신의 원본 클립을 업로드하면, 엔진이 빈칸을 채워 완성된 영상을 제공.
* **한계:** 서버 유지비 및 마케팅 비용 발생, 유저 렌더링 환경(하드웨어 파편화) 통제 난이도 높음.

### 1-2. B2B 비디오 오케스트레이션 API (최종 목표)
* **형태:** API-as-a-Service / Headless Video Engine.
* **구조:** UI/UX를 걷어내고, '템플릿(Yellow) 파싱 기능'과 '다중 에이전트 기반 에셋 매칭 로직(Agent A/B)' 자체를 API 형태로 공급.
* **타깃:** 캡컷(CapCut), 캔바(Canva) 등 대형 편집 플랫폼 또는 대량의 숏폼 생성이 필요한 마케팅 에이전시.
* **장점:** 렌더링 부하를 클라이언트사에 전가할 수 있으며, 트래픽 기반의 안정적인 B2B 과금 모델 구축 가능.

---

## 2. 분석 도메인 및 데이터 처리 확장 루트

### 2-1. 1단계: 일반/정적 도메인 (MVP)
* **대상:** 음식점 홍보, 일상 브이로그, 펫 영상 등 피사체가 명확한 영상.
* **수집 규격:** 비용 최적화를 위해 서버에서 360p~480p로 강제 다운로드.
* **분석 방식:** 시각적 의미(Semantic) 중심. VLM을 통해 "클로즈업", "음식", "먹는 행위" 등의 객체 및 모션 태그를 추출하여 매칭.

### 2-2. 2단계: 게임 및 마이크로 UI 도메인 확장
* **대상:** 킬 로그, 데미지 텍스트, 스킬 쿨타임 등 화면 내 마이크로 픽셀 판독이 필수적인 영상.
* **동적 해상도 라우팅:** 사용자가 입력한 URL의 카테고리(`categoryId=20`)를 식별하여, 게임 도메인인 경우 OCR 및 세밀한 VLM 판독을 위해 720p 이상으로 수집 해상도 상향.
* **오디오 최우선 파이프라인:** 시각 데이터에 앞서 오디오 스트림의 볼륨 피크(타격음, 총소리, 비명 등)를 먼저 추출하여 컷 포인트를 강제 지정하는 로직 추가.

## 3. 인프라 및 렌더링 아키텍처 확장 루트

### 3-1. 1단계: 하드웨어 스니핑 + 브라우저 로컬 렌더링 (MVP)
* **구조:** 백엔드 연산(VLM)을 최소화하기 위해 클라이언트에서 1초 단위 키프레임만 서버로 전송. 최종 조립은 PC 브라우저의 `FFmpeg.wasm`을 사용하여 로컬에서 렌더링.
* **방어 로직:** 접속 시 `navigator.hardwareConcurrency` 및 RAM 사양을 찔러보고, 저사양 기기(똥컴)는 렌더링 OOM(Out of Memory) 방지를 위해 입구에서 차단(Graceful Degradation). 모바일 웹 브라우저 지원 배제.

### 3-2. 2단계: 하이브리드 오프로딩 + 모바일 Native
* **모바일 전환:** iOS/Android의 메모리 한계를 극복하기 위해 AVFoundation / MediaCodec을 활용한 Native 앱으로 전환하여 모바일 시장 진입.
* **하이브리드 렌더링:** 무료 유저는 본인 기기 리소스로 렌더링(로컬). 구독 모델의 VIP 유저나 하드웨어 사양이 미달인 결제자에 한해 원본 영상을 서버로 올려 Serverless GPU 인프라에서 대신 렌더링해 주는 라우팅 적용.

### 3-3. 3단계: WebGPU 기반 클라이언트 엣지 컴퓨팅 (최종 비용 방어)
* **구조:** 서버가 담당하던 VLM(비전 언어 모델) 추론 연산마저 클라이언트 기기로 이전.
* **기술:** WebGPU 및 ONNX Runtime Web을 통해 `moondream2` 수준의 1~2B 파라미터 경량 모델을 사용자의 브라우저/스마트폰 메모리에 직접 로드하여 실행.
* **결과:** 사용자 기기의 그래픽카드로 영상 분석과 렌더링을 모두 수행하게 되어, 서비스 제공자의 **서버 GPU 유지 비용을 물리적으로 0원(Zero)으로 수렴**시킴.
