# 📒 작업 검증 리포트 — 컨벤션

> 작업 **매 구간마다** 이 폴더에 검증 리포트(`.md`)를 1개 남긴다.
> 목적: 각 구간이 **무결성 검증(테스트 green)** 을 통과했음을 시간순으로 증명·추적.

---

## 파일명 규칙

```
YYYY-MM-DD_HHMM_STEP<x>_<슬러그>.md
```

- 날짜·시간이 앞 → 파일 목록이 **자동으로 시간순 정렬**.
- 예: `2026-06-24_2300_STEP3_oracle-and-trunk.md`

## 리포트 템플릿 (이 형식으로 통일)

```markdown
# <STEP id> — <제목>

> <한 줄 요약>

| 항목     | 값               |
| -------- | ---------------- |
| 🕒 일시  | YYYY-MM-DD HH:MM |
| 🔖 커밋  | `hash`           |
| 🎯 RULE  | RULE ...         |
| ✅ 테스트 | N passed         |

## 🎯 무슨 작업

## 📂 변경 파일

## 🔬 무결성 검증   ← pytest 결과(코드블록) + 검증 포인트

## 📝 결과 / 관찰
```

규칙: 헤더 표 4행 고정 · 섹션 이모지·순서 고정 · `🔬 무결성 검증`엔 **반드시 pytest 결과**를 코드블록으로.
